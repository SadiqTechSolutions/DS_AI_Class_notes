🔢 Part 1: Core Data Types

What is a Data Type?

A Data Type is like a label on a box that tells Python what kind of information is inside a variable. This helps Python understand what operations are allowed.

Analogy: You can't add a word to a number, just like you can't pour juice into a toaster. Data types prevent these kinds of errors!

The type() function

To find out the data type of any variable, we use the built-in type() function.

# Create a file named day3_types.py
my_name = "Alex"
print(type(my_name))
# Expected Output: <class 'str'>

Text Type: str

  • Stands for: String
  • Purpose: Stores text data, like names or sentences.
  • Syntax: Use single (') or double (") quotes.
# String examples
first_name = "Sherlock"
last_name = 'Holmes'
quote = "The world is full of obvious things which nobody by any chance ever observes."

Numeric Types: int, float, complex

  • int (Integer): Whole numbers, positive or negative. (e.g., 10, -5, 0)
  • float (Floating-Point Number): Numbers with decimals. (e.g., 9.81, -3.14, 0.0)
  • complex: Numbers with a real and imaginary part (e.g., 3 + 4j). We won't use these often, but it's good to know they exist!
# Numeric examples in day3_types.py
age = 35                  # int
height_meters = 1.82      # float
planet_population = 8000000000 # int

print(type(age))          # Output: <class 'int'>
print(type(height_meters))# Output: <class 'float'>

Boolean Type: bool

  • Stands for: Boolean
  • Purpose: Stores one of two values: True or False.
  • Use Case: Essential for making decisions in code.
# Boolean examples in day3_types.py
is_learning = True
is_tired = False

print(type(is_learning)) # Output: <class 'bool'>

💡 Fun Fact: Boolean logic is named after George Boole, an English mathematician who laid the foundations for the information age in the 1800s!

🗃️ Part 2: Collection Data Types

Collections are used to store multiple items in a single variable.

List: list

  • Analogy: A shopping list. It's ordered, and you can add or remove items.
  • Characteristics: Ordered, Mutable (changeable).
  • Syntax: Square brackets [].
# Create a file named day3_collections.py
# A list of subjects
subjects = ["Data Science", "Python", "AI", "Machine Learning"]
print(subjects)

# Lists are mutable, so we can change them
subjects.append("Statistics") # Add an item to the end
print(subjects)

# Access items by their index (position starts at 0)
print(f"First subject is: {subjects[0]}")

Tuple: tuple

  • Analogy: A framed photograph. The contents are fixed and cannot be changed.
  • Characteristics: Ordered, Immutable (unchangeable).
  • Syntax: Parentheses ().
# A tuple of coordinates in day3_collections.py
home_location = (14.48, 77.59) # Latitude, Longitude
print(home_location)

# Tuples are immutable, this next line would cause an ERROR!
# home_location[0] = 15.0
# TypeError: 'tuple' object does not support item assignment

Dictionary: dict

  • Analogy: A real-world dictionary. You look up a word (the key) to find its definition (the value).
  • Characteristics: Unordered (in older Python versions), Mutable, stores key:value pairs.
  • Syntax: Curly braces {}.
# A dictionary for a student profile in day3_collections.py
student_profile = {
    "name": "John Doe",
    "age": 22,
    "is_enrolled": True,
    "courses": ["Python", "Data Analysis"]
}
print(student_profile)

# Access a value by its key
print(f"Student's name is: {student_profile['name']}")

Set: set

  • Analogy: A bag of unique marbles. There are no duplicates, and the order doesn't matter.
  • Characteristics: Unordered, Mutable, no duplicate elements.
  • Syntax: Curly braces {} (but not for an empty set, that's a dict!).
# A set of unique badge numbers in day3_collections.py
badge_numbers = {101, 102, 103, 102, 104}
print(badge_numbers) # Output will be {101, 102, 103, 104}

🔄 Part 3: Type Conversion & Casting

Sometimes you need to convert a variable from one data type to another.

Type Casting = Explicitly telling Python to convert a value.

  • int(): Converts a value to an integer.
  • float(): Converts a value to a float.
  • str(): Converts a value to a string.
# Create a file named day3_casting.py

# A number stored as a string
age_string = "30"
print(type(age_string)) # Output: <class 'str'>

# Cast the string to an integer to do math
age_number = int(age_string)
print(type(age_number)) # Output: <class 'int'>

# Now we can perform calculations
next_year_age = age_number + 1
print(f"Next year, you will be {next_year_age}")

# Cast an integer to a string to combine it with text
student_id = 123
id_prefix = "STS-"
full_id = id_prefix + str(student_id)
print(full_id) # Output: STS-123

💡 Interesting Facts

  • The float data type in Python is incredibly precise, allowing for scientific calculations.
  • Dictionaries are highly optimized for retrieving data, making them very fast for lookups.
  • The concept of "mutability" is one of the most important and powerful features of Python's data model.
  • Before Python 3.7, dictionaries were officially unordered. Now, they maintain the insertion order of items.
  • A list can hold any other data type as an item, including another list! This is called a nested list.

# Solution for the Day 3 "Personal Data Manager" assignment.

# 1. & 2. Create the user profile dictionary with specified keys and data types.
user_profile = {
    "name": "Alex Smith",
    "age": 29,
    "height_meters": 1.75,
    "is_active_student": True,
    "courses_enrolled": ["Python", "Data Science", "Statistics"],
    "contact_info": ("alex.s@email.com", "123-456-7890"),
    "years_in_school": "3"
}

# 3a. Print the entire profile dictionary.
print("--- Full User Profile ---")
print(user_profile)
print("-" * 25)

# 3b. Print the user's name and the first course they are enrolled in.
# Accessing a value from a dictionary using its key.
# Accessing the first item from a list using its index [0].
print(f"User Name: {user_profile['name']}")
print(f"First Course: {user_profile['courses_enrolled'][0]}")
print("-" * 25)

# 3c. Print the user's email address from the contact info tuple.
# Accessing the first item from a tuple using its index [0].
print(f"Email Address: {user_profile['contact_info'][0]}")
print("-" * 25)

# 3d. Calculate and print the years in school for next year.
# Access the value associated with the "years_in_school" key.
years_str = user_profile["years_in_school"]
# Cast the string value to an integer to perform math.
years_int = int(years_str)
# Calculate next year's value.
next_year_school = years_int + 1
# Print the result in a formatted string.
print(f"Next year, {user_profile['name']} will have been in school for {next_year_school} years.")
print("-" * 25)

# 4. Add a new key "hobbies" and print the updated dictionary.
# Dictionaries are mutable, so we can add new key-value pairs.
user_profile["hobbies"] = ["Reading", "Hiking", "Coding"]
print("--- Updated User Profile with Hobbies ---")
print(user_profile)
