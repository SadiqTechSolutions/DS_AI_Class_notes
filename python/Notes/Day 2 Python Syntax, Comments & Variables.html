🔤 Part 1: Python Syntax

What is Syntax?

Syntax = Rules for writing code that computer can understand

Analogy: Like grammar rules in English - you need proper structure to make sense!

Key Python Syntax Rules:

  1. Indentation = Spaces that show code structure
  1. Comments = Notes that explain code
  1. Case Sensitivity = Name ≠ name

Indentation Example:

# Correct indentation
for i in range(3):
    print(i)  # 4 spaces indented

# Wrong indentation (will cause error)
for i in range(3):
print(i)  # No indentation - ERROR!

💡 Fun Fact: Python uses indentation instead of { } brackets like other languages!

💬 Part 2: Python Comments

Why Comments?

Comments are like sticky notes for your code - they explain what's happening!

Single-Line Comments:

# This is a comment - computer ignores this line
print("Hello World!")  # Comment at end of line

# You can comment out code to disable it
# print("This won't run")
print("This will run")

Multi-Line Comments:

Method 1: Multiple # symbols

# This is line 1 of comment
# This is line 2 of comment
# This is line 3 of comment
print("Code after comments")

Method 2: Triple quotes

"""
This is a multi-line comment
It can span multiple lines
Very useful for long explanations
"""
print("Code after multi-line comment")

📦 Part 3: Python Variables

What are Variables?

Variables are labeled boxes that store information for later use.

Creating Variables:

# No need to declare type - Python figures it out!
name = "Alice"      # String (text)
age = 25           # Integer (whole number)
height = 5.6       # Float (decimal number)
is_student = True  # Boolean (True/False)

Variable Naming Rules:

✅ GOOD Names:

student_name = "John"
age_in_years = 20
_private_var = "secret"
myVariable = "camelCase"

❌ BAD Names:

2names = "error"     # Can't start with number
my-var = "error"     # Can't use hyphens
class = "error"      # Can't use Python keywords

Naming Conventions:

# Snake Case (Recommended for Python)
student_name = "Python"
total_marks = 95

# Camel Case
studentName = "Python"
totalMarks = 95

# Pascal Case
StudentName = "Python"
TotalMarks = 95

🌍 Part 4: Variable Scope

Local Variables:

Variables created inside functions - only work inside that function.

def my_function():
    local_var = "I'm local!"  # Only exists inside function
    print(local_var)

my_function()
# print(local_var)  # ERROR! Can't access outside function

Global Variables:

Variables created in main code - work everywhere.

global_var = "I'm global!"  # Available everywhere

def my_function():
    local_var = "I'm local!"
    print(global_var)   # Can access global variable
    print(local_var)    # Can access local variable

my_function()
print(global_var)       # Can access global variable
# print(local_var)      # ERROR! Can't access local variable
