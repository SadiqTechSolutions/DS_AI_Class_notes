notebooks Part 1: What is a Jupyter Notebook?

A Jupyter Notebook is an interactive web-based application that allows you to create and share documents containing live code, equations, visualizations, and narrative text.

Analogy: Think of a Jupyter Notebook as a digital lab notebook or a scrapbook for your code. It's not just a plain script. You can write a piece of code in one "cell," run it, and see the output (like a table or a chart) immediately below it. Then, in the next cell, you can write notes and explanations about what you just did. It's a perfect blend of code, output, and storytelling.

Why is it essential for Data Science?

  • Interactive Exploration: Run small chunks of code and see results instantly, which is perfect for exploring data.
  • Inline Visualizations: Display charts and graphs directly within your notebook.
  • Documentation: Combine your code with detailed explanations, making your analysis easy to understand and share.
  • Reproducibility: A single notebook file (.ipynb) contains everything needed to reproduce your analysis.

üíª Part 2: Installation and Launch

Jupyter is a package that needs to be installed into a Conda environment. It's a best practice to install it in each new environment where you need it.

Installing Jupyter

Let's use the data-analysis environment we created yesterday.

# 1. Activate the environment you want to work in
conda activate data-analysis

# 2. Install the jupyter package
conda install jupyter

Jupyter Notebook vs. JupyterLab

When you install Jupyter, you get two interfaces:

  • Jupyter Notebook: The classic, original interface. It's simple and focuses on one document at a time.
  • JupyterLab: The next-generation interface. It's more like an IDE (like VS Code) and allows you to work with multiple notebooks, files, and terminals in a tabbed, flexible layout. JupyterLab is recommended for most modern workflows.

Launching JupyterLab

Once installed, you can start the JupyterLab server from your terminal.

# Make sure your 'data-analysis' environment is active
# This command starts the server and opens a new tab in your web browser
jupyter lab

Your terminal will show some logs, and a new browser tab will open, showing you a file browser. This is the JupyterLab interface. Keep the terminal window open; closing it will shut down the server.

ÁïåÈù¢ Part 3: The Jupyter Interface

Cells

The notebook is made up of cells. A cell is a container for either code or text.

  • Code Cells: These are where you write and execute your Python code.
  • Markdown Cells: These are where you write text, headings, lists, and links using Markdown syntax.

The Kernel

The kernel is the "computational engine" that runs your code. When you run a code cell, the notebook sends the code to the kernel, the kernel executes it, and it sends the output back to the notebook. Each notebook has its own kernel.

Command Mode vs. Edit Mode

Your keyboard interacts with a cell in two different modes:

  • Edit Mode (Green border): You are typing inside the cell. Press Enter to enter this mode.
  • Command Mode (Blue border): You are issuing commands that affect the cell itself (like creating new cells, deleting them, etc.). Press Esc to enter this mode.

‚å®Ô∏è Part 4: Essential Keyboard Shortcuts

Working efficiently in Jupyter means using keyboard shortcuts. Make sure you are in Command Mode (blue border) to use these.

ShortcutAction
aInsert a new cell above the current cell.
bInsert a new cell below the current cell.
dd (press twice)Delete the current cell.
mChange the current cell to Markdown.
yChange the current cell to code (Y is near Code).
zUndo cell deletion.
Shift + EnterRun the current cell and select the cell below.
Ctrl + EnterRun the current cell and stay on the same cell.
